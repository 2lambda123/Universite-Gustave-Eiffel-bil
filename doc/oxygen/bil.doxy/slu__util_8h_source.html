149 /*
150  *-- This contains the options used to control the solution process.
151  *
152  * Fact (fact_t)
153  * Specifies whether or not the factored form of the matrix
154  * A is supplied on entry, and if not, how the matrix A should
155  * be factorizaed.
156  * = DOFACT: The matrix A will be factorized from scratch, and the
157  * factors will be stored in L and U.
158  * = SamePattern: The matrix A will be factorized assuming
159  * that a factorization of a matrix with the same sparsity
160  * pattern was performed prior to this one. Therefore, this
161  * factorization will reuse column permutation vector
162  * ScalePermstruct->perm_c and the column elimination tree
163  * LUstruct->etree.
164  * = SamePattern_SameRowPerm: The matrix A will be factorized
165  * assuming that a factorization of a matrix with the same
166  * sparsity pattern and similar numerical values was performed
167  * prior to this one. Therefore, this factorization will reuse
168  * both row and column scaling factors R and C, both row and
169  * column permutation vectors perm_r and perm_c, and the
170  * L & U data structures set up from the previous factorization.
171  * = FACTORED: On entry, L, U, perm_r and perm_c contain the
172  * factored form of A. If DiagScale is not NOEQUIL, the matrix
173  * A has been equilibrated with scaling factors R and C.
174  *
175  * Equil (yes_no_t)
176  * Specifies whether to equilibrate the system (scale A's row and
177  * columns to have unit norm).
178  *
179  * ColPerm (colperm_t)
180  * Specifies what type of column permutation to use to reduce fill.
181  * = NATURAL: use the natural ordering
182  * = MMD_ATA: use minimum degree ordering on structure of A'*A
183  * = MMD_AT_PLUS_A: use minimum degree ordering on structure of A'+A
184  * = COLAMD: use approximate minimum degree column ordering
185  * = MY_PERMC: use the ordering specified by the user
186  *
187  * Trans (trans_t)
188  * Specifies the form of the system of equations:
189  * = NOTRANS: A * X = B (No transpose)
190  * = TRANS: A**T * X = B (Transpose)
191  * = CONJ: A**H * X = B (Transpose)
192  *
193  * IterRefine (IterRefine_t)
194  * Specifies whether to perform iterative refinement.
195  * = NO: no iterative refinement
196  * = SLU_SINGLE: perform iterative refinement in single precision
197  * = SLU_DOUBLE: perform iterative refinement in double precision
198  * = SLU_EXTRA: perform iterative refinement in extra precision
199  *
200  * DiagPivotThresh (double, in [0.0, 1.0]) (only for sequential SuperLU)
201  * Specifies the threshold used for a diagonal entry to be an
202  * acceptable pivot.
203  *
204  * SymmetricMode (yest_no_t)
205  * Specifies whether to use symmetric mode. Symmetric mode gives
206  * preference to diagonal pivots, and uses an (A'+A)-based column
207  * permutation algorithm.
208  *
209  * PivotGrowth (yes_no_t)
210  * Specifies whether to compute the reciprocal pivot growth.
211  *
212  * ConditionNumber (ues_no_t)
213  * Specifies whether to compute the reciprocal condition number.
214  *
215  * RowPerm (rowperm_t) (only for SuperLU_DIST or ILU)
216  * Specifies whether to permute rows of the original matrix.
217  * = NO: not to permute the rows
218  * = LargeDiag: make the diagonal large relative to the off-diagonal
219  * = MY_PERMR: use the permutation given by the user
220  *
221  * ILU_DropRule (int)
222  * Specifies the dropping rule:
223  * = DROP_BASIC: Basic dropping rule, supernodal based ILUTP(tau).
224  * = DROP_PROWS: Supernodal based ILUTP(p,tau), p = gamma * nnz(A)/n.
225  * = DROP_COLUMN: Variant of ILUTP(p,tau), for j-th column,
226  * p = gamma * nnz(A(:,j)).
227  * = DROP_AREA: Variation of ILUTP, for j-th column, use
228  * nnz(F(:,1:j)) / nnz(A(:,1:j)) to control memory.
229  * = DROP_DYNAMIC: Modify the threshold tau during factorizaion:
230  * If nnz(L(:,1:j)) / nnz(A(:,1:j)) > gamma
231  * tau_L(j) := MIN(tau_0, tau_L(j-1) * 2);
232  * Otherwise
233  * tau_L(j) := MAX(tau_0, tau_L(j-1) / 2);
234  * tau_U(j) uses the similar rule.
235  * NOTE: the thresholds used by L and U are separate.
236  * = DROP_INTERP: Compute the second dropping threshold by
237  * interpolation instead of sorting (default).
238  * In this case, the actual fill ratio is not
239  * guaranteed to be smaller than gamma.
240  * Note: DROP_PROWS, DROP_COLUMN and DROP_AREA are mutually exclusive.
241  * ( Default: DROP_BASIC | DROP_AREA )
242  *
243  * ILU_DropTol (double)
244  * numerical threshold for dropping.
245  *
246  * ILU_FillFactor (double)
247  * Gamma in the secondary dropping.
248  *
249  * ILU_Norm (norm_t)
250  * Specify which norm to use to measure the row size in a
251  * supernode: infinity-norm, 1-norm, or 2-norm.
252  *
253  * ILU_FillTol (double)
254  * numerical threshold for zero pivot perturbation.
255  *
256  * ILU_MILU (milu_t)
257  * Specifies which version of MILU to use.
258  *
259  * ILU_MILU_Dim (double)
260  * Dimension of the PDE if available.
261  *
262  * ReplaceTinyPivot (yes_no_t) (only for SuperLU_DIST)
263  * Specifies whether to replace the tiny diagonals by
264  * sqrt(epsilon)*||A|| during LU factorization.
265  *
266  * SolveInitialized (yes_no_t) (only for SuperLU_DIST)
267  * Specifies whether the initialization has been performed to the
268  * triangular solve.
269  *
270  * RefineInitialized (yes_no_t) (only for SuperLU_DIST)
271  * Specifies whether the initialization has been performed to the
272  * sparse matrix-vector multiplication routine needed in iterative
273  * refinement.
274  *
275  * PrintStat (yes_no_t)
276  * Specifies whether to print the solver's statistics.
277  */
